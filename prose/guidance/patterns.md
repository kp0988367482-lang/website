# OpenProse Patterns

Use this file when writing a new `.prose` file.

## Good Patterns

- keep workflows modular
- name sessions clearly
- choose the simplest state mode that fits
- make remote dependencies explicit
- separate orchestration from heavy implementation steps

## Authoring Flow

1. define the workflow goal
2. identify which steps need distinct sessions
3. choose state mode
4. sketch the program structure
5. validate before repeated use
